trace local context cloud

AI agents and coding tools have no memory of what you did between conversations. They start blank every time. Trace solves this by continuously capturing your computer activity, indexing it locally with Supermemory, and exposing it through a chat overlay and API. Your context persists forever.

Why

Every time you ask an AI about your work, it has no idea what you've been doing. The files you edited, the terminal commands you ran, the browser tabs you had open, the project you were focused on. Trace bridges that gap by recording activity as structured events, indexing them into a searchable memory graph, and letting you query everything with natural language. It runs entirely on your machine. Nothing leaves.

How it works
Capture
A persistent PowerShell process streams what you do: foreground window changes, browser URLs (via UI Automation), running processes, and idle state. Separate watchers track file edits, editor projects, and terminal commands. All events are timestamped and tagged with source, app, and project context.
Store
Every event is written immediately to ~/.trace/events.jsonl and asynchronously posted to Supermemory Local on port 6767 for chunking, embedding, and indexing. The local file acts as the source of truth. Supermemory provides hybrid vector-keyword search. If Supermemory is down, all queries fall back to full substring scan of the local file.
Query
An Express server on port 6768 exposes REST endpoints for context retrieval, search, daily summaries, and MCP-compatible tools for AI agents. An LLM-powered Q&A mode sends your query plus relevant memories to any OpenAI-compatible endpoint (OpenRouter, OpenAI, Ollama, etc.) and returns a natural language answer citing specific apps, files, and URLs.
Access
Press Alt+X to open a frameless chat overlay where you can type questions and get answers with supporting evidence. A web-based HUD on port 6769 shows live activity, session timelines, and search results. The API can be used directly, or through MCP tools integrated into OpenCode, Claude Code, Cursor, and other AI coding assistants.
Events captured

Each event carries a source, type, content, timestamp, metadata (app, project, URL, path), and tags for filtering and time bucketing.

  • Foreground app switches with window title
  • Browser URL changes (Chrome, Edge, Brave via UI Automation)
  • Process snapshots every 10 seconds for app open/close detection
  • Idle start and end events after 5 minutes of inactivity
  • File creation and modification events via chokidar
  • Editor project directory detection
  • PowerShell command history capture
  • Auto-generated session summaries every 30 minutes per project
API

All endpoints return JSON. The server runs on port 6768.

GET /context/current - active project, recent events, current session
GET /context/query?q=...&llm=true - search memories with optional AI answer
GET /context/summary - LLM summary of last 2 hours of activity
GET /context/day?date=2026-07-11 - full day with sessions grouped by project
GET /health - service health and Supermemory status
POST /admin/daemon/pause - pause activity capture
POST /admin/daemon/resume - resume activity capture
DELETE /admin/memories - clear all stored events
POST /mcp with tool name and args for AI agent integration
Configuration

Copy .env.example to .env. The server reads these variables at startup:

SUPERMEMORY_URL - http://localhost:6767 (default)
SUPERMEMORY_API_KEY - printed on first supermemory boot
CONTAINER_TAG - namespace for your memories (default: trace)
WATCH_SOURCES - comma-separated: filesystem, browser, editor, terminal
WATCH_PATHS - semicolon-separated directories to monitor
LLM_URL / LLM_MODEL / LLM_API_KEY - for AI-powered Q&A
Supermemory

Trace uses Supermemory Local as its remote memory backend. Supermemory handles chunking, embedding, vector storage, and hybrid search. It runs as a single binary on localhost:6767, inside WSL on Windows. Configure it with your preferred LLM provider for automatic memory extraction:

OPENAI_BASE_URL=https://openrouter.ai/api/v1
OPENAI_API_KEY=sk-or-v1-...
OPENAI_MODEL=tencent/hy3-preview

Supermemory's memory agent workflow extracts facts from each document, enabling semantic search and user profile building over time. When no LLM is configured, documents are still indexed for vector search, but the memory agent is skipped.
