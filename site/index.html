Your memory,
on your machine.

Trace quietly remembers what you do on your computer — the files you touch, the tabs you open, the commands you run — and lets you ask about any of it later in plain language. Nothing leaves your machine unless you point it at an AI.

Why

Every time you ask an AI about your work, it has no idea what you've been doing. The files you edited, the terminal commands you ran, the browser tabs you had open, the project you were focused on. Trace bridges that gap by keeping a private, searchable record of your activity and letting you query it all with natural language. It runs entirely on your machine.

How it works
Capture
A persistent process quietly streams what you do — foreground windows, browser pages, file edits, terminal commands, git activity, and clipboard — all tagged with source, app, and project.
Store
Everything is written to ~/.trace/events.jsonl on your disk and indexed locally with Supermemory for fast, private search. Your data stays yours.
Ask
Press Alt+X to open a calm overlay and ask in plain language — “what was I working on yesterday?” Trace finds the answer and shows its sources.
Access
The same experience is available as a web view, a REST API, and MCP tools you can plug into OpenCode, Claude Code, Cursor, and other AI assistants.
What it remembers

Trace keeps a private, searchable record of your activity so you can find anything later. Sensitive content (passwords, tokens, keys) is redacted automatically.

  • The apps and windows you're working in
  • Browser pages you visit (Chrome, Edge, Brave)
  • Files you create and edit
  • Editor projects and git activity (commits, branches)
  • Terminal commands you run
  • Clipboard snippets — redacted and trimmed for safety
  • Calm, automatic summaries of each session
Intelligence
Dedup
Near-duplicate activity is folded together so your memory stays clean and relevant.
Topics
/context/topics surfaces the themes you've been working on, clustered automatically from recent activity.
Predict
/context/predict proactively surfaces memories and files relevant to whatever you're working on next.
Digest
A quiet daily summary of your day is written to ~/.trace/digests/YYYY-MM-DD.md.
MCP integration

Point your agent at POST /mcp with a tool name and args. Works with OpenCode, Claude Code, Cursor, and any MCP-compatible client.

API

All endpoints return JSON. The server runs locally on port 6768.

GET /context/current — active project, recent activity, current session
GET /context/query?q=...&llm=true — search memories with optional AI answer
GET /context/chat?q=... — free-form AI chat (no activity context)
GET /context/summary?since=... — AI summary of recent activity
GET /context/day?date=2026-07-11 — full day with sessions grouped by project
GET /context/recent-files?limit=20 — most recently touched files
GET /context/project?project=... — all memories for a project
GET /context/timeline?start=ISO&end=ISO — activity in a time range
GET /context/topics?limit=8 — emergent topics from activity
GET /context/predict?project=&path= — proactively relevant memories + files
GET /health — service health and Supermemory status
GET /admin/status — daemon + Supermemory status
POST /admin/daemon/pause — pause activity capture
POST /admin/daemon/resume — resume activity capture
POST /admin/compact — gzip old archives to save disk
DELETE /admin/memories — clear all stored memories
Configuration

Copy .env.example to .env. The server reads these variables at startup:

SUPERMEMORY_URL — http://localhost:6767 (default)
SUPERMEMORY_API_KEY — printed on first Supermemory boot
CONTAINER_TAG — namespace for your memories (default: trace)
WATCH_SOURCES — comma-separated: filesystem, browser, editor, terminal, clipboard
WATCH_PATHS — semicolon-separated directories to monitor
CHROME_HISTORY / EDGE_HISTORY / BRAVE_HISTORY — browser history paths (OS default)
SHELL_HISTORY — terminal history path (PowerShell PSReadLine default)
API_PORT — context API port (default: 6768)
HUD_PORT — web view port (default: 6769)
DIGEST_HOUR — local hour (0–23) to write the daily digest (default: 21)
AUTO_UPDATE_CHECK — poll for newer releases (default: true)
UPDATE_URL — JSON endpoint returning { version, url, notes }
LLM_URL / LLM_MODEL / LLM_API_KEY — for AI-powered Q&A
Supermemory

Trace uses Supermemory Local as its private memory backend. It handles chunking, embedding, and search, and runs entirely on your machine. Configure it with your preferred LLM provider for automatic memory extraction.

Resilience
Watchdog
npm run watchdog runs a supervisor that restarts the server on unexpected exit, with crash-loop protection.
Auto-update
When enabled, trace politely checks for a newer version and lets you know — nothing is forced.
Installer
One-command installers (install.ps1 / install.sh) set up dependencies, configure .env, start Supermemory, and launch trace.
Binaries
Packages as a desktop app via electron-builder, or runs as a plain Node service with npm run watchdog.
