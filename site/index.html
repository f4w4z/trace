trace your memory, on your machine

Trace quietly remembers what you do on your computer: the files you touch, the tabs you open, the commands you run, and lets you ask about any of it later in plain language. Everything stays local. Nothing leaves your machine unless you point it at an AI.

Features
Refined Interface
Sleek, rounded window corners and transparent glassmorphic overlay matching native OS aesthetics.
Daily Summary
Automatic session summarization that digests what you worked on, apps used, and sites visited.
Conversation History
Easily jump back into past logs, chat history, and context streams at any time.
Why

Every time you ask an AI about your work, it starts from scratch. It has no idea what you were just doing. The files you edited, the terminal commands you ran, the tabs you had open, the project you were focused on. Trace quietly keeps a private memory of all of it and lets you ask about it in plain language: “what was I working on yesterday?” It runs entirely on your machine. Nothing leaves.

How it works
Capture
A persistent PowerShell process streams what you do: foreground window changes, browser URLs (via UI Automation), running processes, and idle state. Separate watchers track file edits, editor projects, terminal commands, git activity, and clipboard. All events are timestamped and tagged with source, app, and project context.
Store
Every event is written immediately to ~/.trace/events.jsonl and asynchronously posted to Supermemory Local on port 6767 for chunking, embedding, and indexing. The local file acts as the source of truth. Supermemory provides hybrid vector-keyword search. If Supermemory is down, all queries fall back to full substring scan of the local file.
Query
An Express server on port 6768 exposes REST endpoints for context retrieval, search, daily summaries, and MCP-compatible tools for AI agents. An LLM-powered Q&A mode sends your query plus relevant memories to any OpenAI-compatible endpoint (OpenRouter, OpenAI, Ollama, etc.) and returns a natural language answer citing specific apps, files, and URLs.
Access
Press Alt+X to open a calm chat overlay where you can ask questions and get answers in plain language. A web view on port 6769 offers the same search experience. The API can be used directly, or through MCP tools integrated into OpenCode, Claude Code, Cursor, and other AI coding assistants.
What it remembers

Trace keeps a private, searchable record of your activity so you can find anything later. Sensitive content (passwords, tokens, keys) is redacted automatically.

  • The apps and windows you're working in
  • Browser pages you visit (Chrome, Edge, Brave)
  • Files you create and edit
  • Editor projects and git activity (commits, branches)
  • Terminal commands you run
  • Clipboard snippets: redacted and trimmed for safety
  • Calm, automatic summaries of each session
Intelligence
Dedup
Near-duplicate events (same source + normalized content within 10 min) are dropped before they hit the store or remote.
Topics
/context/topics derives emergent topics from recent activity: frequent keywords, tags, and project names clustered automatically.
Predict
/context/predict proactively surfaces memories and files relevant to a project or file path. Wire it into an editor extension for live context injection.
Digest
At DIGEST_HOUR (local time) a markdown summary of the day is written to ~/.trace/digests/YYYY-MM-DD.md and logged.
Compact
Rotated JSONL archives are gzipped on demand via POST /admin/compact to save disk. Reads transparently decompress .gz files.
MCP integration

Point your agent at POST /mcp with a tool name and args. Works with OpenCode, Claude Code, Cursor, and any MCP-compatible client.

API

All endpoints return JSON. The server runs on port 6768.

GET /context/current: active project, recent events, current session
GET /context/query?q=...&llm=true: search memories with optional AI answer
GET /context/chat?q=...: free-form AI chat (no activity context)
GET /context/summary?since=...: LLM summary of recent activity
GET /context/day?date=2026-07-11: full day with sessions grouped by project
GET /context/recent-files?limit=20: most recently touched files
GET /context/project?project=...: all memories for a project
GET /context/timeline?start=ISO&end=ISO: every event in a time range
GET /context/topics?limit=8: emergent topics clustered from activity
GET /context/predict?project=&path=: proactively relevant memories + files
GET /health: service health and Supermemory status
GET /admin/status: daemon + Supermemory status
POST /admin/daemon/pause: pause activity capture
POST /admin/daemon/resume: resume activity capture
POST /admin/compact: gzip old JSONL archives
DELETE /admin/memories: clear all stored events
POST /mcp · GET /mcp/tools: AI agent integration
Configuration

Copy .env.example to .env. The server reads these variables at startup:

SUPERMEMORY_URL: http://localhost:6767 (default)
SUPERMEMORY_API_KEY: printed on first supermemory boot
CONTAINER_TAG: namespace for your memories (default: trace)
WATCH_SOURCES: comma-separated: filesystem, browser, editor, terminal, clipboard
WATCH_PATHS: semicolon-separated directories to monitor
CHROME_HISTORY / EDGE_HISTORY / BRAVE_HISTORY: browser history paths (OS default)
SHELL_HISTORY: terminal history path (PowerShell PSReadLine default)
API_PORT: context API port (default: 6768)
HUD_PORT: web HUD port (default: 6769)
DIGEST_HOUR: local hour (0–23) to write the daily digest (default: 21)
AUTO_UPDATE_CHECK: poll UPDATE_URL for newer releases (default: true)
UPDATE_URL: JSON endpoint returning { version, url, notes }
LLM_URL / LLM_MODEL / LLM_API_KEY: for AI-powered Q&A
Supermemory

Trace uses Supermemory Local as its remote memory backend. Supermemory handles chunking, embedding, vector storage, and hybrid search. It runs as a single binary on localhost:6767, inside WSL on Windows. Configure it with your preferred LLM provider for automatic memory extraction:

OPENAI_BASE_URL=https://openrouter.ai/api/v1
OPENAI_API_KEY=sk-or-v1-...
OPENAI_MODEL=google/gemma-4-31b-it:free

Supermemory's memory agent workflow extracts facts from each document, enabling semantic search and user profile building over time. When no LLM is configured, documents are still indexed for vector search, but the memory agent is skipped.

Resilience
Watchdog
npm run watchdog runs a supervisor that restarts the server on unexpected exit. Crash-loop protection: max 5 restarts per minute.
Auto-update
When AUTO_UPDATE_CHECK=true and UPDATE_URL is set, trace polls for a newer version and logs the result. Set AUTO_UPDATE_DOWNLOAD=true to also fetch the release asset into ./release.
Installer
install.ps1 (Windows) and install.sh (WSL/Linux) install deps, build, configure .env, start Supermemory, and launch trace under the watchdog in one command.
Binaries
The Electron overlay packages via electron-builder (npm run dist:app, produces release/Trace Setup*.exe). The server can be run via node dist/index.js or npm run watchdog.
